## Scripts

### `./start.sh`

On **first run** generates `APP_KEY`, `DB_PASSWORD`, `DB_ROOT_PASSWORD`, and `IN_PASSWORD`, writes them back into `.env`, then pulls images and starts the stack. Subsequent runs skip secret generation.

```bash
./start.sh
```

First-run initialisation (~90 seconds):

1. MySQL comes up and passes its healthcheck
2. `in-app` runs Laravel migrations and seeds the database
3. Creates the admin account from `IN_USER_EMAIL` / `IN_PASSWORD`
4. `in-nginx` starts serving once `in-app` is healthy

### `./stop.sh`

Stops all containers. Volumes are preserved.

```bash
./stop.sh
```

### `./teardown.sh`

Interactive full teardown вЂ” lists everything that will be removed, prompts for confirmation, then deletes all containers, volumes, images, and networks.

```bash
./teardown.sh
```

To re-install completely fresh (new secrets):

```bash
sed -i 's/^APP_KEY=.*/APP_KEY=GENERATE_ME/' .env
sed -i 's/^DB_PASSWORD=.*/DB_PASSWORD=GENERATE_ME/' .env
sed -i 's/^DB_ROOT_PASSWORD=.*/DB_ROOT_PASSWORD=GENERATE_ME/' .env
sed -i 's/^IN_PASSWORD=.*/IN_PASSWORD=GENERATE_ME/' .env
./teardown.sh
./start.sh
```

## Files

| File / Dir | Purpose |
|---|---|
| `.env` | All runtime configuration and secrets |
| `docker-compose.yml` | 4-container stack definition |
| `nginx/invoiceninja.conf` | Invoice NinjaвЂ“specific Nginx location rules |
| `nginx/laravel.conf` | Laravel default Nginx server block |
| `php/php.ini` | PHP runtime settings (memory, upload limits, etc.) |
| `php/php-fpm.conf` | PHP-FPM pool configuration |
| `supervisor/supervisord.conf` | Supervisord config вЂ” runs PHP-FPM and the Laravel queue worker together in `in-app` |
| `start.sh` | Start / first-run setup |
| `stop.sh` | Stop (volumes preserved) |
| `teardown.sh` | Full wipe with confirmation |

### `.env` вЂ” values of interest

| Variable | Value | Notes |
|---|---|---|
| `IN_TAG` | `latest` | Docker image tag; pin to a version (e.g. `5.13.18`) for reproducibility |
| `LISTEN_PORT` | `8092` | Host port |
| `APP_URL` | `http://localhost:8092` | Must match the browser URL; changing after first run requires a DB update |
| `APP_KEY` | *(generated)* | Laravel encryption key вЂ” **never change** after first run; invalidates all encrypted tokens |
| `IN_USER_EMAIL` | `admin@example.com` | Initial admin email; only consumed on first start |
| `IN_PASSWORD` | *(generated)* | Initial admin password; only consumed on first start |
| `DB_PASSWORD` | *(generated)* | MySQL application user password |
| `DB_ROOT_PASSWORD` | *(generated)* | MySQL root password |
| `PDF_GENERATOR` | `snappdf` | Uses bundled Chromium for PDF generation вЂ” no external dependency |
| `MAIL_MAILER` | `log` | Emails are written to container logs by default; set to `smtp` for real delivery |

## Architecture

```
Browser
  в””в”Ђв–є localhost:8092
        в””в”Ђв–є in_nginx  (Nginx 1.27)
              в”њв”Ђв–є /public/*  в†’  in_public volume  (static assets, served directly)
              в””в”Ђв–є /*.php     в†’  in_app:9000  (PHP-FPM)
                                  в”њв”Ђв–є in_db:3306   (MySQL 8)
                                  в””в”Ђв–є in_redis:6379 (Redis 7 вЂ” sessions, cache, queue)
```

**Containers:**

| Container | Image | Role |
|---|---|---|
| `in_db` | `mysql:8.0` | Database |
| `in_redis` | `redis:7-alpine` | Sessions, cache, and job queue |
| `in_app` | `invoiceninja/invoiceninja-debian:latest` | PHP-FPM app + queue worker |
| `in_nginx` | `nginx:1.27-alpine` | Web server and reverse proxy |

**Volumes:**

| Volume | Contents |
|---|---|
| `in_mysql` | MySQL data |
| `in_redis` | Redis persistence |
| `in_public` | Laravel `public/` directory (shared between `in-app` and `in-nginx`) |
| `in_storage` | Laravel `storage/` directory (uploads, logs, cache, invoices) |

## Cheat Sheet

### Logs

```bash
# All services
docker compose -p invoiceninja logs -f

# App container (PHP-FPM + queue worker)
docker logs in_app -f

# Web server
docker logs in_nginx -f

# Database
docker logs in_db -f
```

### Shell access

```bash
docker exec -it in_app bash
docker exec -it in_db mysql -u ninja -p ninja   # prompts for DB_PASSWORD
```

### Run artisan commands

```bash
docker exec in_app php artisan <command>

# Common examples:
docker exec in_app php artisan cache:clear
docker exec in_app php artisan config:clear
docker exec in_app php artisan ninja:check-data
docker exec in_app php artisan queue:restart
```

### Reset admin password

```bash
docker exec in_app php artisan ninja:reset-password --email=admin@example.com
```

### Back up data

```bash
# Database dump
docker exec in_db mysqldump -u ninja -p"$(grep '^DB_PASSWORD=' .env | cut -d= -f2-)" ninja > invoiceninja_backup.sql

# Storage volume (PDFs, uploads, logs)
docker run --rm -v in_storage:/data -v $(pwd):/backup alpine \
  tar czf /backup/invoiceninja_storage.tar.gz /data
```

### Upgrade Invoice Ninja

> **Important:** Read the release notes before upgrading. Some versions require specific migration steps.

1. Update `IN_TAG` in `.env` to the target version (e.g. `5.13.18`)
2. `./stop.sh && ./start.sh`
3. Migrations run automatically on container start

### Configure outgoing email (SMTP)

Edit `.env`:

```
MAIL_MAILER=smtp
MAIL_HOST=smtp.yourprovider.com
MAIL_PORT=587
MAIL_USERNAME=your@email.com
MAIL_PASSWORD=yourpassword
MAIL_ENCRYPTION=tls
MAIL_FROM_ADDRESS=invoices@yourdomain.com
```

Then restart: `./stop.sh && ./start.sh`

### Configure payment gateways

Done via the web UI: **Settings в†’ Payment Settings в†’ Connect** (Stripe, PayPal, etc.)

## Debugging

### App not starting (healthcheck failing)

```bash
docker logs in_app --tail 50
docker logs in_nginx --tail 20
```
Check that the database is healthy before `in-app` starts.

### PDF generation broken

```bash
# Check snappdf is working
docker exec in_app php artisan ninja:pdf-test
```

If it fails, check that Chromium is installed inside the container:

```bash
docker exec in_app which chromium || docker exec in_app which chromium-browser
```

### Queue worker not processing jobs

```bash
docker exec in_app php artisan queue:restart
docker logs in_app | grep -i queue
```

### APP_URL mismatch (redirect loops)

If the browser redirects to the wrong URL after login, update `APP_URL` in `.env` and clear the config cache:

```bash
docker exec in_app php artisan config:clear
docker exec in_app php artisan cache:clear
./stop.sh && ./start.sh
```
